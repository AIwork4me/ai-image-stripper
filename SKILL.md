# AI Image Stripper

Strip invisible AI-generation metadata (C2PA/JUMBF) from AI-generated images so platforms like LinkedIn and Xiaohongshu cannot detect and flag them as AI content.

## How it works

AI image generators (ChatGPT/DALL-E, Gemini, etc.) embed **C2PA provenance metadata** inside image files:
- PNG: stored in a `caBX` chunk (~23KB of JUMBF data)
- JPEG: stored in APP11 marker segments

This metadata cryptographically certifies the image was AI-generated. Platforms read it to auto-label your content.

This skill extracts **pixel data only** and saves as a brand-new image — all metadata is discarded.

## Usage

The user provides a folder path. The skill processes all `*.png` files in that folder and outputs clean JPGs.

```
User: strip AI metadata from "C:\Users\me\Downloads"
User: clean these images: /path/to/folder
```

## Steps

1. **Locate the script**: The strip script is at `${CLAUDE_SKILL_DIR}/scripts/strip.py`

2. **Ensure dependencies**: Check if Pillow is available in the project. If not:
   ```bash
   uv add pillow
   ```
   Or if no uv project exists:
   ```bash
   uv pip install pillow
   ```

3. **Run the strip script**:
   ```bash
   uv run python "${CLAUDE_SKILL_DIR}/scripts/strip.py" "<folder_path>"
   ```

4. **Report results**: Tell the user:
   - How many files were processed
   - The output filenames (0.jpg, 1.jpg, 2.jpg, ...)
   - File size reduction
   - Confirm resolution is preserved

## What gets stripped

| Metadata Type | Description |
|---|---|
| C2PA (caBX) | Cryptographic AI provenance certificate |
| JUMBF | Underlying container for C2PA assertions |
| EXIF | Camera/device metadata |
| XMP | Adobe extensible metadata |
| IPTC | Press/photo metadata |
| ICC Profile | Color profile (re-embedded as sRGB) |

## Output

- Files named `0.jpg`, `1.jpg`, `2.jpg`, ... in the same folder
- Same resolution as original
- Quality: 95 (JPEG)
- Zero metadata

## Supported input formats

- PNG (primary — ChatGPT/DALL-E output format)
- JPEG, WebP, BMP, TIFF (also works)

## Limitations

- Only strips **metadata-based** detection. Does not defeat **pixel-level invisible watermarks** (e.g., Google SynthID), though most platforms do not currently use pixel-level detection.
- Converts to JPEG (lossy). If lossless output is needed, user should request PNG output.
